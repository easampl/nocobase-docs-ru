# Обзор системы контекста

Система контекста движка потоков NocoBase разделена на три уровня, каждый из которых соответствует своей области видимости. Корректное использование позволяет гибко разделять и изолировать сервисы, конфигурации и данные, улучшая сопровождаемость и масштабируемость.

- **Глобальный контекст (`FlowEngineContext`)**: глобально уникален, доступен всем моделям и потокам, подходит для регистрации глобальных сервисов, конфигураций и т.д.
- **Контекст модели (`FlowModelContext`)**: используется для совместного использования контекста внутри дерева моделей. Подмодели автоматически делегируют в контекст родительской модели, поддерживаются одноимённые переопределения. Подходит для логики уровня модели и изоляции данных.
- **Контекст выполнения потока (`FlowRuntimeContext`)**: создаётся при каждом запуске потока и существует на протяжении всего цикла выполнения. Подходит для передачи данных, хранения переменных и фиксации статуса выполнения внутри потока. Поддерживает два режима: `mode: 'runtime' | 'settings'`, то есть режим выполнения и режим настроек.

Все контексты — контекст движка потоков (`FlowEngineContext`), контекст модели потока (`FlowModelContext`), контекст выполнения потока (`FlowRuntimeContext`) и т.д. — являются подклассами или экземплярами контекста потока (`FlowContext`).

---

## 🗂️ Диаграмма иерархии

```text
FlowEngineContext (Глобальный контекст)
│
├── FlowModelContext (Контекст модели)
│     ├── Sub FlowModelContext (Подмодель)
│     │     ├── FlowRuntimeContext (Контекст выполнения потока)
│     │     └── FlowRuntimeContext (Контекст выполнения потока)
│     └── FlowRuntimeContext (Контекст выполнения потока)
│
├── FlowModelContext (Контекст модели)
│     └── FlowRuntimeContext (Контекст выполнения потока)
│
└── FlowModelContext (Контекст модели)
      ├── Sub FlowModelContext (Подмодель)
      │     └── FlowRuntimeContext (Контекст выполнения потока)
      └── FlowRuntimeContext (Контекст выполнения потока)
```

- Контекст модели (`FlowModelContext`) может обращаться к свойствам и методам глобального контекста (`FlowEngineContext`) через механизм делегирования, обеспечивая общий доступ к глобальным возможностям.
- Подмодель контекста модели (`FlowModelContext`) может обращаться к контексту родительской модели (синхронная связь) через механизм делегирования, поддерживая одноимённые переопределения.
- Асинхронные parent-child модели не создают делегирующую связь, чтобы избежать загрязнения состояния.
- Контекст выполнения потока (`FlowRuntimeContext`) всегда обращается к своему контексту модели (`FlowModelContext`) через делегирование, но не распространяет изменения вверх.

## 🧭 Режим выполнения и режим настройки

Контекст выполнения потока (`FlowRuntimeContext`) поддерживает два режима, различаемых параметром `mode`:

- `mode: 'runtime'` (режим выполнения): используется во время фактического выполнения потока. Свойства и методы возвращают реальные данные. Например:
  ```js
  console.log(runtimeCtx.steps.step1.result); // 42
  ```

- `mode: 'settings'` (режим настройки): используется на этапе проектирования и настройки потока. Доступ к свойствам возвращает шаблонную строку переменной, что упрощает работу с выражениями и выбор переменных. Например:
  ```js
  console.log(settingsCtx.steps.step1.result); // '{{ ctx.steps.step1.result }}'
  ```

Такая двухрежимная архитектура обеспечивает доступность данных в режиме выполнения и удобную генерацию ссылок на переменные и выражения в режиме настройки, повышая гибкость и удобство движка потоков.

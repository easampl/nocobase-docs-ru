# Поток событий

В движке потоков все интерфейсные компоненты **управляются событиями**.
Поведение, взаимодействие и изменения данных компонентов запускаются событиями и выполняются через поток.

## Статический поток и динамический поток

В движке потоков потоки можно разделить на два типа:

### **1. Статический поток**

- Определяется разработчиками в коде;
- Применяется ко **всем экземплярам класса модели**;
- Обычно используется для обработки общей логики класса модели;

### **2. Динамический поток**

- Настраивается пользователями в интерфейсе;
- Действует только для конкретного экземпляра;
- Обычно используется для персонализированного поведения в конкретных сценариях;

Кратко: **статический поток — это шаблон логики, заданный на классе, а динамический поток — персонализированная логика, заданная на экземпляре.**

## Правила связывания и динамический поток

В системе конфигурации движка потоков есть два способа реализации событийной логики:

### **1. Правила связывания**

- Это **инкапсуляция встроенных шагов потока событий**;
- Проще в настройке и более семантичны;
- По сути это всё равно упрощённая форма **потока событий**.

### **2. Динамический поток**

- Полные возможности конфигурации потока;
- Настраиваются:
  - **Триггер (`on`)**: определяет момент запуска;
  - **Шаги выполнения (`steps`)**: определяют исполняемую логику;
- Подходят для более сложной и гибкой бизнес-логики.

Следовательно, **правила связывания ≈ упрощённый поток событий**, и их базовые механизмы совпадают.

## Согласованность действия потока

И **правила связывания**, и **потоки событий** должны использовать один и тот же набор **действий потока**.
То есть:

- **Действие потока** определяет действия, которые может вызывать поток;
- Оба подхода используют единую систему действий, а не две отдельные;
- Это обеспечивает переиспользование логики и единообразное расширение.

## Концептуальная иерархия

Концептуально, базовая абстрактная связь в модели потока выглядит так:

```bash
FlowModel
 └── FlowDefinition
      ├── FlowEventDefinition
      │     ├── Глобальные события
      │     └── Локальные события
      └── FlowActionDefinition
            ├── Глобальные события
            └── Локальные события
```

### Описание иерархии

- **Модель потока (FlowModel)**
  Представляет сущность модели с настраиваемой и исполняемой логикой потока.

- **Определение потока (FlowDefinition)**
  Определяет полный набор логики потока (включая условия запуска и шаги выполнения).

- **Определение события потока (FlowEventDefinition)**
  Определяет источник запуска потока, включая:
  - **Глобальные события**: например запуск приложения, завершение загрузки данных;
  - **Локальные события**: например изменение полей, клики по кнопкам.

- **Определение действия потока (FlowActionDefinition)**
  Определяет исполняемые действия потока, включая:
  - **Глобальные действия**: например обновление страницы, глобальные уведомления;
  - **Локальные действия**: например изменение значений полей, переключение состояния компонентов.

## Итог

| Концепт | Назначение | Область |
|------|------|-----------|
| **Статический поток** | Логика потока, заданная в коде | Все экземпляры XXModel |
| **Динамический поток** | Логика потока, заданная в интерфейсе | Один экземпляр модели потока (FlowModel) |
| **Событие потока** | Определяет триггер (когда запускать) | Глобальная или локальная |
| **Действие потока** | Определяет логику выполнения | Глобальная или локальная |
| **Правило связывания** | Упрощённая инкапсуляция шагов потока событий | Уровень блока и действия |
